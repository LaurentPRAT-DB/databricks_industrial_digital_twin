How what-if works in the repo
1 · Clone the twin model
Create a parallel RDF graph namespace (e.g. :scenario_A) in the Delta table. All triples are copied; only the modified parameter triples are overwritten.
2 · Inject parameter overrides
SPARQL UPDATE or PySpark writes new sensor threshold triples. E.g. change dt:setpoint for spindle speed on :CNCMachine_01 without touching the live twin.
3 · Run simulation / ML inference
Feed the scenario graph into AutoML-trained predictive models (vibration → bearing life, temp deviation → defect rate). Results land in a scenario-scoped Delta table.
4 · Compare in the app
The Databricks App can render multiple graph namespaces side-by-side. Metrics like OEE, MTBF, and defect rate are diffed against the live twin baseline.
